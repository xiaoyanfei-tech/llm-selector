# Services

`llm-selector` is open source. If you need a business-ready recommendation, the same framework can be used as a consulting workflow.

## 1. LLM Selection Report

For small and medium-sized businesses choosing a model stack.

Deliverables:

- business scenario analysis
- model shortlist and trade-off table
- cost and deployment considerations
- data-sensitivity and compliance risk review
- recommended PoC plan
- verification checklist

Best for:

- companies choosing between GLM, Qwen, DeepSeek, Kimi, Claude, GPT, and local models
- teams preparing their first AI adoption plan
- founders who need a practical decision memo rather than a benchmark paper

## 2. AI Coding Stack Recommendation

For teams adopting AI coding tools.

Deliverables:

- Claude Code / Cursor / Cline / Continue fit analysis
- model and gateway recommendation
- endpoint verification plan
- developer rollout plan
- cost and security checklist

Best for:

- engineering teams adopting AI coding assistants
- companies using GLM-compatible or OpenAI-compatible gateways
- teams that need to verify the actual model behind a gateway

## 3. Endpoint Verification Report

For teams that already have a model gateway but want to know whether it is trustworthy.

Deliverables:

- claimed model vs observed behavior
- tokenizer / metadata / context-window evidence
- routing and downgrade risk review
- final verdict and remediation suggestions

Best for:

- GLM-5.2 gateway verification
- OpenAI-compatible gateway acceptance testing
- AI vendor due diligence

## 4. PoC Implementation Plan

For teams ready to test a real use case.

Deliverables:

- 2-4 week PoC scope
- success metrics
- model/provider choices
- integration checklist
- data-handling policy
- pilot-user rollout plan

## How to request help

Open an issue using the `Business LLM selection request` template and include:

- company type / industry
- target scenario
- data sensitivity
- budget range
- preferred deployment model
- existing tools and constraints

Do not include API keys, private URLs, customer records, or confidential documents in public issues.
